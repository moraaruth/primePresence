## 📋 DEPLOYMENT CHECKLIST

### 1. Set Up Supabase Database (REQUIRED - Do This First!)

```bash
1. Go to https://app.supabase.com
2. Select your Prime Presence project
3. Click "SQL Editor" → "New Query"
4. Copy entire content from: supabase/migrations/001_init_saas_schema.sql
5. Paste into SQL editor
6. Click "Run"
```

**Verify it worked:**
```sql
SELECT COUNT(*) FROM sites;
-- Should return: 0
```

### 2. Add Environment Variables

Update `.env.local` with:

```bash
# You probably already have these
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=xxxxx
SUPABASE_SERVICE_ROLE_KEY=xxxxx

# ADD THIS (required for AI generation)
OPENAI_API_KEY=sk-xxxxxxxx

# ADD THESE (for later Stripe integration)
STRIPE_SECRET_KEY=sk_test_xxxxxx
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_xxxxxx
```

### 3. Install Dependencies

```bash
npm install
```

(Most should already be installed; this ensures everything is there)

### 4. Test Locally

```bash
npm run dev
```

Visit:
- `http://localhost:4028` - Homepage (new split design)
- `http://localhost:4028/signup` - Sign up
- `http://localhost:4028/onboarding` - AI brief
- `http://localhost:4028/dashboard` - Site management
- `http://localhost:4028/builder/[siteId]` - Site editor

### 5. Deploy to Vercel

```bash
git add .
git commit -m "Add SaaS layer: DIY builder + dashboard"
git push
```

Vercel will:
- Build the Next.js app
- Deploy to production
- Use environment variables from Vercel dashboard

## 🚨 IMPORTANT NOTES

### Subdomain Routing

**Local Development:**
- Sites are viewable at `http://localhost:4028/:slug`
- Test: `/tech-startup`

**Production (Vercel):**
- Need to configure wildcard domain in Vercel
- Set up: `*.primepresence.site` → Points to your Vercel deployment
- Then sites will be at: `tech-startup.primepresence.site`

**Setup on Vercel:**
1. Go to your Vercel project
2. Settings → Domains
3. Add `primepresence.site` and `*.primepresence.site`
4. Follow DNS instructions

### Authentication

- Signup creates user in `auth.users` (Supabase)
- Trigger auto-creates `user_profiles` record
- Trigger auto-creates free `subscription` with 14-day trial
- Session stored in httpOnly cookie

### AI Generation

- Uses OpenAI Claude 3.5 Sonnet
- Generates site structure, pages, and content
- Takes 30-60 seconds
- Stores in database with all sections

## 🛠️ TROUBLESHOOTING

### "Site not found" on dashboard
- Check user is authenticated
- Check Supabase RLS policies are applied
- Run: `SELECT * FROM sites;` in Supabase SQL editor

### AI generation hangs
- Check OPENAI_API_KEY is set
- Check API quota in OpenAI dashboard
- Check internet connection

### Published site shows wrong content
- Verify site status is "published" in DB
- Check sections are loaded (SQL: `SELECT * FROM sections WHERE page_id = '...'`)
- Clear browser cache

### Subdomain not working
- Verify DNS is configured on Vercel
- Check subdomain value in `sites` table
- Test via direct URL: `/[subdomain-value]`
